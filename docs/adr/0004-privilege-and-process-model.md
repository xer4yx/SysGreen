# Privilege and process model: non-elevated by default, on-demand elevated helper

SysGreen's UI and Tray Agent run **non-elevated**. Actions that don't require admin (per-user `HKCU\...\Run`, the user's Startup folder — where most RAM-hog apps actually live) execute directly. When a batch includes actions that *do* require admin (`HKLM\...\Run`, Windows Services, certain scheduled tasks), the UI spawns a **short-lived elevated helper** that performs the batch, writes the undo record, and exits. The app detects per-action whether elevation is needed and batches elevated actions so the user sees **at most one UAC prompt per Apply**.

We rejected (a) running the whole app/tray agent elevated — it forces a UAC prompt at every login or an insecure auto-elevation, and keeps a fat always-resident privileged process; and (b) an always-on elevated Windows Service broker — a permanent privileged background service, ironic for a tool meant to remove background services and an unnecessary attack surface. Model 3 is the only option consistent with both the "lightweight" and "safe/reversible" promises: no boot-time elevation, no permanent privileged process, minimal attack surface.
