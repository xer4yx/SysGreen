# First-run flow and SysGreen's own autostart entry

**First run:** (1) Welcome + privacy disclosure (the consent screen from ADR-0012, launch tracking local-only with an off switch present); (2) seed habit history from UserAssist/Prefetch (ADR-0008) and scan + classify all Manageable Items, with progress shown; (3) land on the Recommendations view with a useful ranked list (no cold start), nothing pre-selected and nothing auto-applied (ADR-0007) — education over pressure.

**SysGreen registers its own Autostart Entry** for the Tray Agent, because the habit engine must observe launches at every logon, even when the UI is closed. This is deliberately ironic (a footprint manager adding a startup item) and is handled with full transparency rather than hidden: the Agent is kept genuinely tiny (a sampling launch-logger, ADR-0004), SysGreen's own entry **appears in the All Items list like any other and is user-disableable** (with a note that disabling it stops habit tracking and degrades recommendations to static-only), and it is **never concealed**. The rejected alternative — running the Agent only while the UI is open — would cripple habit collection. Eating our own dog food (transparent, disableable, listed) is what preserves the trust the product depends on.
